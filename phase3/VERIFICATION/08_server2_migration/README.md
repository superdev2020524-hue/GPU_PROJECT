# 08 - Server 2 Migration

## Purpose

Prepare a controlled path from Server 1 engineering success to Server 2
client-facing mediated vGPU deployment.

## Scope

- keep Server 2 stable for demonstrations until mediated mode is ready;
- define criteria for switching Server 2 to mediated vGPU mode;
- create deployment checklist;
- create rollback checklist;
- create client demonstration script;
- verify that the demo does not depend on hidden manual state.

## Closure Criteria

- Server 1 passes required general gates;
- Server 2 can deploy the same mediated stack;
- rollback to stable mode is documented;
- client-facing verification is repeatable.
